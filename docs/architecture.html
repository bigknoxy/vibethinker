Components

  • CLI vibecli.py

    8 commands: prompt, chat, code, think, explain, sh, review, eval. Retry logic with exponential backoff (5s, 10s, 20s) and server health polling via /v1/models.

  • Server serve_vibethinker.py

    FastAPI server running on port 8002. Exposes POST /v1/chat/completions (OpenAI-compatible) and GET /v1/models. Spawns a worker subprocess per request for crash isolation.

  • Worker vibe_worker.py

    Isolated subprocess that loads the GGUF model, generates a response via llama-cpp-python, and exits. If the model segfaults (a known issue after ~12-15 requests), the server stays alive.

  • Watchdog watchdog.sh

    Shell script that polls /v1/models every 10 seconds. If 3 consecutive health checks fail, it restarts the server. Catches crashes faster than systemd alone (~30s detection + 10s reload).

  • Model Qwen2-3.1B-Q8_0.gguf

    Self-converted Q8_0 GGUF, 3.1B parameters, 3.1 GB file size. Qwen2 architecture. Community GGUFs were broken, so we converted from the HuggingFace source using llama.cpp/convert.py.

  • Systemd Service vibethinker.service

    Systemd unit with Restart=on-failure. Ensures the server survives boot and recovers from unexpected exits.

Data Flow

vibe prompt "list all pdfs in /tmp" # CLI command → chat() # builds OpenAI-style payload → POST /v1/chat/completions # HTTP to FastAPI → serve_vibethinker.py # parses request, spawns worker → vibe_worker.py # loads GGUF into memory → llama-cpp-python Python API # CPU inference → Qwen2-3.1B-Q8_0 GGUF model # generates response → <think>reasoning</think>answer # structured output → response JSON ⟶ CLI stdout # printed to terminal
