CLI & API Reference

VibeThinker provides 8 CLI commands plus an OpenAI-compatible HTTP API. All commands connect to the server at $VIBE_API (default: http://localhost:8003).

CLI Commands

vibe prompt

Send a single prompt and get a response. The primary command for one-shot queries.

vibe prompt <text> [-t temp] [-m max_tokens] [-s] [--no-think] [-o file]
-t, --temp <float>
Temperature (default: 0.5)
-m, --max-tokens <int>
Maximum output tokens (default: 512)
-s, --silent
Suppress progress output; print only the response
--no-think
Strip the <think> reasoning block from output
-o, --output <file>
Write response to file instead of stdout
# Explain a git command $ vibe prompt "explain what git rebase -i HEAD~3 does" --no-think Interactive rebase lets you edit, squash, reorder, or drop the last 3 commits before they are applied on top of HEAD.

vibe chat

Multi-turn chat. Maintains conversation history for the session.

vibe chat <text> [-S system] [-t temp] [-m max_tokens] [-s] [--no-think]
-S, --system <text>
System message for the conversation
-t, --temp <float>
Temperature (default: 0.5)
-m, --max-tokens <int>
Maximum output tokens (default: 512)
-s, --silent
Suppress progress output
--no-think
Strip the reasoning block from output
# Start a chat session $ vibe chat "why is my docker container exiting immediately?" \ -S "you are a docker expert" --no-think Common causes: the ENTRYPOINT/CMD process exits, no foreground process is running, or there's an error in your entrypoint script. Run `docker logs <container>` to see the exit reason.

vibe code

Generate code for a given task. Optimized for code output.

vibe code <task> [-t temp] [-m max_tokens] [-s] [-o file]
-t, --temp <float>
Temperature (default: 0.3 — lower for code)
-m, --max-tokens <int>
Maximum output tokens (default: 1024)
-s, --silent
Suppress progress output
-o, --output <file>
Write generated code to file
# Generate a Rust Fibonacci function and save it $ vibe code "write a fibonacci function in rust with memoization" -o fib.rs

vibe think

Show only the model's reasoning chain. Strips the answer, keeps the <think> block.

vibe think <text> [-t temp] [-m max_tokens] [--json]
-t, --temp <float>
Temperature (default: 0.5)
-m, --max-tokens <int>
Maximum output tokens (default: 1024)
--json
Output reasoning as JSON
# See how the model reasons about sorting algorithm choice $ vibe think "which sorting algorithm should I use for a nearly-sorted array of 1M integers?" <think>The user has a nearly-sorted array of 1 million integers. Insertion sort is O(n) on nearly-sorted data but O(n^2) average case. Timsort is O(n) on nearly-sorted and O(n log n) worst case — it's the clear choice here. Python uses Timsort natively in list.sort().</think>

vibe explain

Explain a file or code snippet. Reads the file and sends it to the model for explanation.

vibe explain <file> [-t temp] [-m max_tokens] [-s] [-o file]
-t, --temp <float>
Temperature (default: 0.5)
-m, --max-tokens <int>
Maximum output tokens (default: 1024)
-s, --silent
Suppress progress output
-o, --output <file>
Write explanation to file
# Explain a Python file $ vibe explain src/main.py --no-think This file implements a FastAPI server with two endpoints. /v1/chat/completions accepts POST requests with OpenAI- compatible payloads and delegates to a subprocess worker.

vibe sh

Generate a shell command for a task. Output is a single command ready to run.

vibe sh <task> [-t temp] [-m max_tokens]
-t, --temp <float>
Temperature (default: 0.3 — lower for commands)
-m, --max-tokens <int>
Maximum output tokens (default: 256)
# Generate a shell command $ vibe sh "find all pdfs in /tmp modified in the last 24 hours" find /tmp -name '*.pdf' -mtime -1

vibe review

Review a file for bugs, style issues, and improvements.

vibe review <file> [-t temp] [-m max_tokens]
-t, --temp <float>
Temperature (default: 0.5)
-m, --max-tokens <int>
Maximum output tokens (default: 1024)
# Review a Python file $ vibe review src/vibecli.py 1. Missing error handling in `chat()` when server is down 2. Hardcoded timeout of 30s — should be configurable 3. Line 142: variable `responce` is a typo of `response`

vibe eval

Run the eval suite against the model. Measures pass rate across 27 tasks.

vibe eval [suite] [-n trials] [-v] [--report]
suite
Specific suite to run (default: all suites)
-n, --trials <int>
Number of trials per task (default: 1)
-v, --verbose
Show per-task results as they run
--report
Generate an HTML report
# Run the full eval suite $ vibe eval --report -v [code/python_fib] PASS (0.34s) [code/rust_fib] PASS (0.41s) [sh/find_pdfs] PASS (0.22s) [sh/git_log] FAIL (0.31s) ... 27/28 passed (96.4%) Report written to eval_report.html

HTTP API

The server exposes an OpenAI-compatible API. Any OpenAI client can use VibeThinker by changing the base URL.

POST /v1/chat/completions

Generate a completion from a chat-formatted prompt. Fully compatible with the OpenAI Chat Completions schema.

POST http://localhost:8003/v1/chat/completions
Content-Type: application/json

{
  "model": "vibethinker",
  "messages": [
    { "role": "user", "content": "explain git rebase" }
  ],
  "temperature": 0.5,
  "max_tokens": 512,
  "stream": false
}

Response:

{
  "id": "cmpl-abc123",
  "object": "chat.completion",
  "created": 1712345678,
  "model": "vibethinker",
  "choices": [{
    "index": 0,
    "message": {
      "role": "assistant",
      "content": "<think>explanation</think>answer"
    },
    "finish_reason": "stop"
  }],
  "usage": {
    "prompt_tokens": 12,
    "completion_tokens": 142,
    "total_tokens": 154
  }
}

GET /v1/models

List available models. Used by the CLI for health checking and by OpenAI clients to verify connectivity.

GET http://localhost:8003/v1/models

{
  "object": "list",
  "data": [
    {
      "id": "VibeThinker",
      "object": "model"
    }
  ]
}

GET /stats

Request statistics: total requests, errors, cache hits/misses, and cache size.

GET http://localhost:8003/stats

{
  "requests": 42,
  "errors": 0,
  "total_time": 580.2,
  "cache_hits": 18,
  "cache_misses": 12,
  "cache_size": 9
}

POST /feedback

Submit feedback for self-improvement data collection.

POST http://localhost:8003/feedback
Content-Type: application/json

{
  "rating": 5,
  "comment": "helped with my regex"
}

Response: {"status": "recorded"}
